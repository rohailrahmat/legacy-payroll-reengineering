Architecture Recovery and Model-Driven Re-Engineering of Legacy Software Systems for Modernization

A comprehensive case study applying systematic software re-engineering, static analysis, and design pattern migrations to transform a vulnerable monolithic PHP Payroll System into a modern, cloud-native TypeScript microservices application.

Re-Engineering Goals Accomplished

This project provides the complete analytical baseline for software modernization. It recovers documentation for an undocumented legacy program, categorizes technical debt using Martin Fowler's taxonomies, establishes clean, side-by-side refactored code demonstrations, and plots a realistic Strangler Fig-based roadmap. Critical Severity Refactoring: Extract Class Location: payroll.php (2,000+ lines)

The entire system—employee database, payroll calculations, leave logging, and HTML generation—is crammed inside a single global script. This violates the Single Responsibility Principle.

Legacy PHP (Vulnerable Monolith) payroll.php
Refactored TypeScript (NestJS Service) PayrollService.ts
